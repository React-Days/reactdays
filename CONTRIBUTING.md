# Contributing to ReactDays Ecosystem ⚛️

First off, thank you for considering contributing! It’s people like you who will make **ReactDays** a success.

## How Can I Contribute?

### 🛠️ Code Contributions
- Check our [Roadmap](./ROADMAP.md) for upcoming features.
- Look for "good first issue" labels in our Issues tab.
- Fork the repo, create a branch (`feature/my-feature`), and submit a Pull Request.

### 📝 Documentation & Design
- We need help refining our schemas and documentation.
- If you are a designer, we are currently looking for UI/UX input for the Event Dashboard.

### 📢 Spread the Word
- Share [reactdays.com](https://reactdays.com) with your local React meetup groups.

## Development Process
1. **Fork** the repository.
2. **Commit** your changes with descriptive messages.
3. **Ensure** your code follows our TypeScript/Linting rules.
4. **Submit** a PR and link the related issue.

*By contributing, you agree that your code will be licensed under the MIT License.*
